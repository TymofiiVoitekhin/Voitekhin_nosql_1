## Частина 1. Схема даних та трансформація

Після виконання `02_transform.js`, сирі дані були перетворені у колекцію `tracks`. 
**Приклад фінального документа:**
```json
{
  "_id": "64b1f...",
  "track_id": "5SuOikwiRyPMVoIQDJUgSV",
  "track_name": "Comedy",
  "album_name": "Comedy",
  "explicit": false,
  "popularity": 73,
  "duration_ms": 130000,
  "track_genre": "acoustic",
  "artists": ["Gen Hoshino"],
  "duration_sec": 130.0,
  "popularity_tier": "high",
  "audio_features": {
    "danceability": 0.676,
    "energy": 0.461,
    "tempo": 87.917,
    // ... інші характеристики
  }
}

Відповіді на питання (Частина 1)

Чому аудіо-характеристики винесені в окремий об’єкт audio_features?
Це логічне групування (Embedding), яке зменшує забруднення кореневого документа та дозволяє витягувати всі характеристики однією проєкцією (audio_features: 1). Таке вкладення вигідне, коли поля логічно пов'язані та зазвичай зчитуються разом. Створює проблеми лише тоді, коли рівень вкладеності стає занадто великим (MongoDB лімітує вкладеність) або якщо потрібно будувати складні індекси на масиві глибоко вкладених об'єктів.

Чому виконавці зберігаються як масив, а не як рядок?
Плоский рядок зі знаком ; унеможливлює швидкий та точний пошук, вимагаючи дорогих Regex-запитів. Масив дозволяє базі використовувати Multikey Indexes. Це кардинально спрощує запити: знайти трек конкретного артиста можна через пряме співставлення find({ artists: "Queen" }), а порахувати аналітику — використавши оператор $unwind.

Що таке $out і чим він відрізняється від $merge?
$out повністю перезаписує вказану цільову колекцію (видаляє стару і створює нову). Ідеально підходить для повних пакетних трансформацій (як у нашому завданні).
$merge дозволяє вставляти результати агрегації в існуючу колекцію: він може оновлювати поточні документи або додавати нові (upsert). Використовується для інкрементальних оновлень даних.
